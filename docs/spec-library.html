Library — extension spec

All Library features not in v1: full layout (metadata panel, source tree, sort/search/multi-select), duplicate-by-content detection, XMP sidecars, missing-file relink, and the deferred organising/batch list. v1 ships import + grid + filmstrip only (see v1 §13).

Depends on v1 §13 (basic Library) and §14 (software stack). This spec extends the v1 Library module in place — same catalog, same import flow, same source-of-truth model.

  • Left — source list: "All Photos" virtual node + a folder tree mirroring the on-disk paths of imported roots. No albums/keywords (see §6). Selecting a source filters the grid.
  • Centre — thumbnail grid: adjustable thumbnail size, sort by Capture date (default) / Import date / File name, with ascending/descending. Multi-select per §2.
  • Right — metadata panel: read-only — filename, full path, pixel dimensions, camera, lens, focal length, ISO/shutter/aperture (where EXIF provides), capture date, import date. Editable IPTC is deferred (see §6).
  • Top toolbar: Import button (v1 §13.4), sort dropdown, view-size control, filename search box.
  • Bottom — filmstrip: shared with the Edit module (v1 §13.2); shows the current source's photos in order; the loaded/selected photo is highlighted.

2. Browse & select (extended)

  • Multi-select: Ctrl/Cmd-click toggles individual photos; Shift-click extends a range. The selection persists across module switches and is the foundation for batch operations (§6).
  • Search: the toolbar search box filters by filename (substring, case-insensitive) within the current source. Advanced filters (date/camera/lens/etc.) are deferred (§6).
  • Sort: Capture date (default) / Import date / File name, with ascending/descending.

3. Duplicate detection (by content)

An optional "detect duplicates by content" checkbox in the import dialog (off by default, for speed) marks same-content-different-path files with a "duplicate" badge and excludes them by default. The v1 import dialog only dedupes by path; this adds the content-hash path.

Open: the catalog's optional content-hash check from v1 §13.1 is the storage-side counterpart of this. Confirm whether the content hash is stored at import when this checkbox is on, vs computed on demand.

4. XMP sidecars

  • A Write XMP sidecars command exports the current edit list to a .xmp-style file beside the original for interop/backup.
  • Read XMP on import is an import-dialog checkbox (off by default) that seeds a photo's edit list from an existing sidecar.
  • Sidecars are never the source of truth; the DB is.

5. Missing files & relink

Because files are referenced in place, a file moved/renamed/deleted outside the app becomes missing:

  • Missing photos show a ? badge on the thumbnail and are greyed in the grid; the metadata panel shows "File not found at <path>."
  • Per-photo relink: a "Find missing file" command opens a file picker; choosing the new location updates the catalog path for that photo.
  • Folder relink: a "Find missing folder" command on a missing folder in the source tree re-roots all its descendant paths in one action (the common case — a whole shoot moved).
  • Missing photos keep their edit list and previews; they are not deleted from the catalog automatically.

6. Deferred (organising & batch)

  • Albums / collections / smart collections.
  • Keywords / tags / ratings / flags / colour labels.
  • Faces / people; GPS / map view.
  • Batch & multi-photo operations (hole #8): apply a sheet to many photos, copy/paste edits, Lightroom's "previous edit" / sync, paste a single sheet layer vs the whole stack. The §2 multi-select model is the foundation these build on.
  • Advanced filters: by date range, camera, lens, focal length, ISO, flag/rating (once those exist).
  • Folder-watch / auto-import.
  • Tethered capture.
  • Preview-size policy & on-demand re-render tiers (implementation detail).
  • Editable metadata / IPTC editing.
  • Multi-catalog support / catalog switching / catalog backup & restore UI.

Hole #8 — Multi-photo & batch operations

#8 — Multi-photo & batch operations med

Can you select multiple photos and apply a sheet to all? Copy/paste edits between photos (Lightroom's "previous edit" / sync)? Paste a single sheet layer vs the whole stack? Not specified in v1. Note: the multi-select model these operations need is defined in §2; the batch operations themselves remain deferred (§6) and will be specced in a later pass.
