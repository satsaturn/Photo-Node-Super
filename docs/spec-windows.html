Windows integration — extension spec

The Windows shell integration dropped from v1: "Open with Photo-Node-Super" shell verb across 12 image/RAW extensions, drag-drop from Explorer (single/multi-file/one-level folder recursion), and the list of what remains out of scope. v1 keeps high-DPI, file dialogs, path handling, keyboard, and storage locations (v1 §14.7); only the shell-integration extras live here.

Depends on v1 §14.7 (Windows concerns kept in v1) and §13 (import flow — drag-drop reuses the import candidate list). High-DPI, native file dialogs, path handling, keyboard shortcuts, and the per-user storage locations under %APPDATA%\Photo-Node-Super\ / %LOCALAPPDATA%\Photo-Node-Super\ stay in v1 §14.7; this extension adds shell-verb registration, Explorer drag-drop recursion, and the lens pack storage directory (which arrives with spec-lens).

1. Explorer integration

  • Drag-drop from Explorer into the Library grid — single file, multi-file, or one-level folder recursion over supported extensions — is in scope for this extension. (v1 only has the Import button; this adds drag-drop as an entry point to the same import flow in v1 §13.4.)
  • A "Open with Photo-Node-Super" shell verb is registered for common image and RAW extensions: .jpg, .jpeg, .png, .tif, .tiff, .dng, .cr2, .cr3, .nef, .arw, .raf, .orf, .rw2.

2. Out of scope for this extension

  • Explorer thumbnail provider.
  • Preview handler.
  • "Edit" context-menu verb.
  • Jump lists.

3. Lens pack storage location

Arrives with spec-lens:

  • User-installed lens packs: %APPDATA%\Photo-Node-Super\lens-packs\.

This slots into the storage-locations list in v1 §14.7 (which in v1 lists catalog DB, preview cache, node sheets, and logs only).
