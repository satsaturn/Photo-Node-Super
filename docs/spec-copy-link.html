Copy/Link — extension spec

Adds the copy/link choice on add, copy snapshots, the sheet-file picker, the Pin (global panel slot), the per-sheet Copy/Link default override, and the linked-sheet broken-link / versioning UX. Not in v1; v1 ships custom sheets as linked-only with no prompt and no picker/pin.

Depends on v1 §3 (add flow), §5 (layer header baseline), §6 (phase system), §12 (sheet settings panel), §13 (catalog stores edit list + linked-sheet path/phase). This spec layers copy-link behaviour on top of the v1 linked-only baseline.

1. Copy / link prompt on add

In v1, pressing + in the Custom Edits section creates a linked layer with no prompt. This extension restores the prompt:

  • Picking/creating a sheet fills the panel with that sheet's UI-node controls.
  • The app then asks: copy or link?
    • Copy = safe snapshot; the sheet's current state is frozen into the photo's edit data. Later edits to the sheet file do not affect this layer.
    • Link = live to the sheet file; editing the file updates every linked layer that references it.
  • The choice is asked each time, unless the sheet sets a Copy/Link default override (§3) that skips the prompt.

2. Layer header controls added

Restores the two layer-header controls dropped from v1 §5:

  • Sheet-file picker — dropdown to swap which file this layer points to. (v1 workaround: remove the layer and re-add.) Swapping triggers the copy/link prompt for the new file, respecting the file's override.
  • Pin icon → global panel slot (stays visible across all photos & restarts; slider values still stored per photo).
Open (Hole #10): Pin × copy/link interaction. Pin = "global panel slot, stays visible across all photos; slider values stored per photo." But if a pinned sheet was added as a copy (per-photo), does the pinned slot show that one photo's copy, or the sheet file's defaults? And for a linked pinned sheet, do all photos share one slider-value set or per-photo values? The "stored per photo" clause conflicts with "linked = live to the file." Needs a clear rule — see Holes below.

3. Sheet settings: copy/link default

Restores the §13.2 (v1 §12.2) bullet dropped from v1:

  • Copy/Link default override — global default is "ask each time" on add; a sheet can opt to default to copy or link so it skips the prompt.

The override lives on the sheet file's settings panel (v1 §12.2). The same mechanism is mirrored in spec-lens §3.1 for lens profile files' copyLinkDefault field.

4. Phase-lock & copy-freeze semantics

Restores the phase paragraphs dropped from v1 §6:

  • Phase lives on the sheet file ⇒ a linked layer's phase is locked to the file (duplicate the file to run the same sheet in a different phase).
  • A copied layer freezes the phase at copy time. Editing the sheet file's phase later does not change the copy's phase.

5. Linked-sheet broken links & versioning

From v1 §13.7 (the linked-sheet broken-link UX was deferred here):

  • A linked layer whose sheet file is missing/moved shows a broken-link badge in the layer header and offers relink. This parallels the photo missing-file/relink flow in spec-library §5.
  • Versioning / migration (Hole #12): editing a linked sheet updates all linked photos. What if a sheet previously had a "Warmth" slider and a new version removes it — do photos that had a Warmth value break? Is there a migration path, a "frozen at last-compatible version" fallback, or just graceful degradation? Unspecified.

Holes #10–#12

#10 — Pin × copy/link interaction med

Pin = "global panel slot, stays visible across all photos; slider values stored per photo." But if a pinned sheet was added as a copy (per-photo), does the pinned slot show that one photo's copy, or the sheet file's defaults? And for a linked pinned sheet, do all photos share one slider-value set or per-photo values? The "stored per photo" clause conflicts with "linked = live to the file." Needs a clear rule.

#11 — Linked-sheet deletion / broken links med

If a user deletes (or moves/renames) a sheet file from their folder, what happens to photos that link to it? Broken-link UI in the layer header (§5)? Auto-convert to copy? Greyed-out panel? Not specified beyond the §5 badge + relink affordance.

#12 — Linked-sheet versioning / migration med

Editing a linked sheet updates all linked photos. What if v1 had a "Warmth" slider and v2 removes it — do photos that had a Warmth value break? Is there a migration path, a "frozen at last-compatible version" fallback, or just graceful degradation? Not specified.
