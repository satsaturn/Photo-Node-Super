Photo-Node-Super v1 — Style mockups

10 visual style directions for the Edit module. Each is a fully self-contained HTML file rendering the same spec-v1 content (Library | Edit | Nodes | Lens switcher, photo canvas, histogram, global/mask/crop tabs, Basic/Colour/HSL/Tone Curve/Detail built-ins, custom-add, shared filmstrip) in a different style. All are Win 11 themed. See spec-v1 for the layout each one implements.

Section 2 — built with the frontend-design skill

10 additional style directions · each anchored to a concrete aesthetic reference · distinctive Google Fonts

A second pass, produced using the agent's frontend-design skill. Each direction is anchored to a specific reference (a magazine, an era, a piece of software, a material) rather than a generic adjective. Typography is committed — not Inter, not Roboto, not a purple-on-white gradient — and the implementation is intentionally opinionated. The spec-v1 content is identical across all 20 mockups; only the visual language differs.

Section 3 — clean "normal plane" directions with Zen-style grain frontend-design

5 grounded, refined variations · subtle gradient mesh + fine grain · curated typographic pairings

A third pass, also produced with the frontend-design skill. These are deliberately grounded — not maximalist or experimental. Each uses a clean modern font pair (Geist, Inter, Plus Jakarta Sans, DM Sans, Sora) and a carefully toning palette, set on a subtle gradient mesh with fine SVG grain. The Zen Browser reference is the through-line: calm, restrained, real-feeling.

How to use this folder. Each file is standalone — open any in a browser to view at full size. The content shown in every mockup is the same Edit module per spec-v1 §3 (single-result canvas + right panel with histogram + global/mask/crop tabs + built-in edit list + custom-add) plus the shared filmstrip and 4-way module switcher from §13.2. Differences are pure visual style.

Shortlisted pick. Mockup 10 (Photo-first / Focus) is the current "normal plane" pick — the most production-ready starting point if you want a clean, minimal, photo-first layout. Open it from the callout box at the top of the page.

Three sections. Section 1 (mockups 01–10) are baseline Win 11 look-and-feel style directions — conservative system-aligned treatments using Segoe UI Variable. Section 2 (mockups 11–20) are characterful aesthetic directions produced via the agent's frontend-design skill — committed typographic pairings, anchored to concrete references, intentionally opinionated. Section 3 (mockups 21–25) is a refined "normal plane" set — clean, with nice color toning and a Zen Browser-style grain background — produced via the same frontend-design skill.

Suggested next step. Use mockup 10 as the base. Pick a section 3 palette to skin it (21 lavender, 22 indigo, 23 terracotta, 24 slate, 25 dusk) and we can iterate on a refined v1 mockup that fuses the chosen base with the v1 spec's data-type palette, node editor, and Library module.
