Settings — extension spec

Placeholder. The Settings UI is not in v1; v1 ships a gear icon in the header that opens an empty dialog pointing here. This spec lists the categories we plan to cover, with no depth — to be revisited later.

Depends on v1 §15 (Settings stub), §13 (catalog/storage), §14.7 (Windows keyboard + storage locations), §12.4 (panel-split persistence).

1. Purpose

Settings will be the central place for user-tunable preferences: themes, panel density, storage paths, performance knobs, keyboard rebinding, and so on. v1 ships baked-in defaults (current dark theme, current panel density, the storage locations from v1 §14.7, the keyboard shortcuts from the same section). The Settings dialog itself is reserved as a stub in v1 so the gear icon is visible from day one and the framework is in place.

This spec is intentionally thin — the items below are general ideas of what will live here, not decisions. We will revisit each one in depth later.

2. Planned categories

  • Themes — dark/light toggle, accent colour picker, high-contrast option.
  • UI density & typography — compact / comfortable / cosy panel density; font-size scaling; icon size.
  • Storage & catalog management — show/change catalog DB path, preview cache location, node-sheet folder, lens-pack folder (when spec-lens ships). Backup/restore catalog. Multi-catalog switching (when spec-library ships).
  • Performance & rendering — preview render quality, cache size limit + clear cache, toggle thumbnail generation on import, CPU thread budget. Ties into spec-gpu later.
  • Keyboard shortcuts editor — view/rebind the shortcuts v1 §14.7 and v1 holes #21 touch on.
  • Default phase / new-sheet preferences — default phase for new sheets (v1 §6), default copy/link behaviour (when spec-copy-link ships), default mask shape (when spec-masks ships).
  • EXIF / metadata display prefs — which EXIF fields show in the metadata panel (when spec-library ships), units (mm/in), date format.
  • Workspace & window prefs — remember last module, remember panel-split proportions (already in v1 §12.4), remember window size/position, filmstrip size.
